# ankiweb Sub-project — Editor/Reviewer completeness (close the dead-control gaps)

**Status:** design (2026-06-03). Triggered by a live-CDP audit (workflow `w3fy6dx8z`) that found controls the reused `editor.js`/`reviewer.js`/SvelteKit frontends render but ankiweb never wired. User approved: implement Fields + Cards + Preview, plus Reviewer action menu, Editor attach-media, Card Info page, and Graphs→Browser deep-link.

## Goal
Make the reused frontends' visible controls actually work, by reusing existing backend ops + rebuilding the Qt-only dialogs as server-rendered pages (the E4/E5/I3 pattern). No sync, no add-ons. Out of scope (documented no-ops): record-audio (needs browser MediaRecorder), and the Qt-profile-persistence cosmetics (lastTextColor/Highlight, setTagsCollapsed, toggleSticky-persist, shrink/mathjax/closetags toggles) — their in-editor effect already works; only the remembered preference is lost.

## Key mechanism (verified live)
The editor pages (`/add`, `/edit`) load `bootstrap.js` and connect the WS bridge, so `editor.js`'s toolbar buttons DO send `bridgeCommand("fields")` / `"cards"` / `"preview"` / `"attach"` over the WS to the `editor`/`add` context — they currently hit no handler branch (silent no-op). We can either handle them server-side or intercept client-side. **Decision:** intercept client-side in an injected `editor_links_js()` (sibling of `paste_handler_js()`), because Fields/Cards/Preview should open as a SEPARATE page (keeping the editor intact, mirroring Anki's separate dialogs) via `window.open`, and `attach` needs a client-side `<input type=file>`. The screens themselves are new routes.

## Decomposition (dependency-ordered; each its own plan → implement → review → merge)
- **F1 — Card Info page** (trivial): the `card-info` SvelteKit bundle is already vendored, `card_stats`/`get_review_logs` are already PASSTHROUGH, `card-info` is in `SVELTEKIT_PAGES`. Add `GET /card-info/{ids:path}` to `build_sveltekit_router` (mirrors `change-notetype/{ids:path}`; route nodes are `[cardId]` + `[cardId]/[previousId]`). Entry: a "Card Info" link in the browser detail pane (reviewer entry comes with F7). Test: route 200 + the SPA boots (Playwright optional).
- **F2 — Graphs→Browser deep-link** (small): the served SvelteKit pages have NO `pycmd` bridge (raw index.html), so the graphs count-links' `browserSearch:<q>` goes nowhere. Serve the SvelteKit index for `/graphs` (and the other SPA pages) with a tiny injected inline bridge: `window.pycmd = window.bridgeCommand = c => { if (c.startsWith("browserSearch:")) location.href = "/browse?q="+encodeURIComponent(c.slice(14)); }`. Add `?q=` support to `GET /browse` (prefill + auto-search). Keeps `_anki/` RPCs (those are fetch, unaffected).
- **F-editor-links — editor trigger glue** (small; ships with F4): `editor_links_js()` injected into `/add` + `/edit` wraps `window.bridgeCommand` to intercept `attach` (→ hidden file-input → POST `/upload_media` → `pasteHTML(<img>/[sound:])`, reusing the paste path), `fields` (→ `window.open('/fields/'+ntid)`), `cards` (→ `window.open('/card-layout/'+ntid)`), `preview` (→ `window.open('/preview/'+nid)`), passing through everything else to the real bridge. Needs the page to expose `window.__ankiwebNotetypeId` (+ existing `__ankiwebEditNid`); the editor/add load handler already knows both.
- **F3 — Editor attach-media** (📎): the `attach` branch of `editor_links_js` + reuse `/upload_media`. (Folds into F-editor-links; tested via the editor page.)
- **F4 — Preview** (`/preview/{nid}`): a read-only page rendering each of the note's cards' question+answer HTML using the card render machinery (reuse `reviewer.py`'s `render_question`/`render_answer` building blocks; verify the exact `col`/`Card` render API at plan time — `card.render_output()` / `col.render_existing_card`). Reuses `reviewer.css` + MathJax. Card switcher if the note has multiple cards. No scheduling, no mutation.
- **F5 — Fields editor** (`/fields/{ntid}`): server-rendered form (E4/E5/I3 pattern) listing the notetype's fields with add / rename / reposition (up/down) / delete, the sort-field radio, and per-field font/size/RTL/description. Backend = `col.models` field ops (the AnkiConnect `modelFieldAdd/Remove/Rename/Reposition/SetFont/SetFontSize/SetDescription` already wrap them — reuse those code paths). Submit over WS (`savefields:`) → mutate the notetype dict → `update_notetype`/`update_dict` via `run_op` (broadcast) → `ankiwebPrefsError`-style `#err` on failure. New context `fields`.
- **F6 — Card-template editor** (`/card-layout/{ntid}`): per-template `qfmt`/`afmt` textareas + a shared `css` textarea + add/remove/rename/reposition template, plus a **live preview pane** reusing F4's render. Backend = `col.models` template/css ops (AnkiConnect `updateModelTemplates`/`updateModelStyling`/`modelTemplateAdd/Remove/Rename/Reposition`). Submit (`savelayout:`) → `update_dict` via `run_op`. New context `cardlayout`. (Live preview re-renders via an RPC or on save.)
- **F7 — Reviewer action menu** (largest): add a reviewer actions affordance (a "More" menu/bar + keyboard shortcuts) to `reviewer_page_body`, and branches in `make_reviewer_handler` for: Mark note (`*` — add/remove `marked` tag, redraw `#_mark` via `_drawMark`), Flag (Ctrl+1–7 / clear — `col.set_user_flag_for_cards`, redraw `#_flag` via `_drawFlag`), Bury card/note (`-`/`=`), Suspend card/note (`@`/`!`), Set Due (prompt → `col.sched.set_due_date`), Forget (`col.sched.schedule_cards_as_new`), Delete note (Ctrl+Del → `col.remove_notes` + advance), Undo (`u` → `col.undo`), Card Info (`i` → open `/card-info/{cid}`, completing F1's reviewer entry). All ops exist (most wrapped in `ankiweb/ankiconnect/actions/cards.py`/`notes.py`). Card-mutating actions that change the queue re-fetch the next card; mark/flag redraw in place.

Ship order: **F1 → F2 → F4 (+F-editor-links) → F3 → F5 → F6 → F7.** (Cheap/independent first; F6's preview reuses F4; F7 completes F1's reviewer entry.)

## Cross-cutting
- New routes registered in `build_screen_router` (server-rendered: /fields, /card-layout, /preview) or `build_sveltekit_router` (/card-info); new WS contexts (`fields`, `cardlayout`) in `register_screen_handlers`.
- i18n: all new server-rendered labels use `tr.<key>()` from `ankiweb.i18n` (the I2 convention) — verify keys per plan; keyless ankiweb-specific strings stay English.
- Tests: each feature gets handler/render unit tests + a Playwright boot/interaction test where it exercises the reused frontend.
- Mutating handlers broadcast via `run_op` so other open screens refresh.

## Risks
F6 live-preview + F5/F6 notetype mutation must use the cached-dict + `update_dict` pattern (per the B3 notetype learnings) or changes won't persist. F7 is large and touches the working reviewer — guard every branch against a null/!current card (the existing reviewer pattern). The editor-links interceptor must pass through the real bridge commands (blur/key/saveTags) untouched. The SvelteKit bridge injection (F2) must not break the SPA's own `_anki/` RPC fetches.
